# Changelog Guide

Use this subtree for `docs/changelog/**`.

Read `docs/ai/doc-governance-standard.md` first. This folder is the dated closeout trace for meaningful rounds. It is not a second source of roadmap, goal, structure, or plan truth.

## Rules

1. Daily entries belong in `CHANGELOG_YYYY-MM-DD.md`.
2. One daily file may contain multiple sections for separate rounds.
3. Each section should include `Changed`, `Why`, `Validation`, and `Modified Files`.
4. If a round changes a canonical doc or executable plan status, update that owning doc in the same round before logging it here.
5. Keep file paths repo-relative and validation commands copy-pastable.
