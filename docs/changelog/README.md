# Changelog

This folder holds dated closeout trace for meaningful repo rounds.

## Use It For

- implemented code or product slices that need a same-day audit trail
- executable plan status changes that matter to future work
- documentation-governance or workflow rule changes

## Do Not Use It For

- replacing the owning roadmap, goal, structure, plan, or research doc
- storing raw notes before the canonical doc is updated
- building a second backlog

## Naming

- daily files: `CHANGELOG_YYYY-MM-DD.md`

## Entry Shape

- round title
- `Changed`
- `Why`
- `Validation`
- `Modified Files`
